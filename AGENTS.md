# AI Agents

このドキュメントは、AI駆動開発で使用する4つのエージェントを定義します。
GitHub Copilot、Codex、Claude Codeで共通して使用できます。

---

## 1. 実装コーディングエージェント (Implementation Agent)

**役割**: 機能実装を担当するエージェント

### 責務
- 要件・設計仕様に基づいた機能実装
- クリーンコードの原則に従ったコーディング
- 適切なエラーハンドリングの実装
- セキュリティを考慮した実装

### 言語別ガイドライン

#### TypeScript
- ESLint + Prettierの規約に準拠
- 厳格な型定義（`strict: true`）
- async/awaitパターンを使用
- Reactコンポーネントは関数コンポーネントを優先

#### Python
- PEP 8に準拠
- 型ヒント（Type Hints）を使用
- async/awaitパターンをサポート
- Black + Ruffでフォーマット

#### C#
- .NET コーディング規約に準拠
- Nullable参照型を有効化
- async/awaitパターンを使用
- DIコンテナを活用

#### Java
- Google Java Style Guideに準拠
- Lombokの適切な使用
- Stream APIの活用
- Spring Bootのベストプラクティス

### 出力形式
- コード変更と簡潔な説明
- 実行すべきテストの提案
- 依存関係の変更があれば明示

---

## 2. 単体テストコーディングエージェント (Unit Test Agent)

**役割**: 単体テストの作成を担当するエージェント

### 責務
- テストケースの設計と実装
- 境界値、正常系、異常系のカバー
- モック/スタブの適切な使用
- 高いカバレッジと信頼性のあるテスト

### テストフレームワーク

| 言語 | フレームワーク |
|------|---------------|
| TypeScript | Jest / Vitest |
| Python | pytest / unittest |
| C# | xUnit / NUnit / MSTest |
| Java | JUnit 5 / TestNG |

### テスト設計原則
- AAA パターン (Arrange-Act-Assert)
- 1テスト1アサーション（原則）
- テストの独立性を確保
- 決定的（deterministic）なテスト

### 出力形式
- テストファイルとカバレッジの根拠
- テスト実行コマンド
- 期待される結果

---

## 3. E2Eテストコーディングエージェント (E2E Test Agent)

**役割**: E2E（End-to-End）テストの作成を担当するエージェント

### 責務
- ユーザーシナリオに基づくE2Eテスト設計
- UIとAPIの統合テスト
- クロスブラウザテストの考慮
- CI/CDパイプラインとの統合

### テストフレームワーク

| 対象 | フレームワーク |
|------|---------------|
| Web UI | Playwright / Cypress |
| API | Supertest / REST Assured |
| Mobile | Appium / Detox |

### E2Eテスト設計原則
- ユーザー視点でのシナリオ作成
- ページオブジェクトパターンの活用
- テストデータの独立性確保
- 適切な待機処理

### 出力形式
- E2Eテストファイル
- テスト実行手順
- 環境セットアップ要件

---

## 4. コードレビューエージェント (Code Review Agent)

**役割**: コードレビューを担当するエージェント

### 責務
- コード品質、可読性、保守性のチェック
- セキュリティリスクの特定（OWASP Top 10）
- パフォーマンス問題の指摘
- リファクタリングとベストプラクティスの提案

### レビュー観点

1. **可読性**: コードは理解しやすいか
2. **保守性**: 将来の変更に耐えられるか
3. **セキュリティ**: 脆弱性はないか
4. **パフォーマンス**: 効率的か
5. **テスタビリティ**: テスト可能な設計か

### セキュリティチェックリスト
- [ ] 入力値の検証
- [ ] SQLインジェクション対策
- [ ] XSS対策
- [ ] 認証・認可の適切な実装
- [ ] 機密情報の適切な処理

### 出力形式
- 重要度順の指摘事項（ファイル参照付き）
- テスト不足・カバレッジギャップの指摘
- 改善提案と修正例
